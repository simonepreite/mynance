## 2. What went well

- **Money-correctness discipline never regressed across 6 epics.** Pure calc
  modules (`app/calc/`: money, liquidita, secchiello, periodo, allocazione,
  patrimonio, ricorrenza) with no DB/IO, worked-example unit tests, and a
  `test_purity.py` guard that fails the build if the package ever imports a
  DB/FastAPI dependency. Integer cents + derived-on-read held everywhere.
- **Per-Utente isolation by construction.** Every owned entity goes through the
  `UserScopedRepository` choke point; cross-Utente access is a 404 with no data
  leak. New entities (Categorie → Movimenti → Secchielli → Riconciliazioni →
  Patrimonio → Regole) inherited isolation for free.
- **Contract stayed aligned.** Regenerating the typed OpenAPI client on every
  backend change let `tsc` catch drift immediately (e.g. the
  `renameCategoria`→`updateCategoria` rename surfaced as a build error, not a
  runtime bug).
- **Tight increments.** One commit + one story doc + sprint-status update per
  story/epic, CI watched green before moving on. Easy to reason about, easy to
  revert.
- **Elegant lazy generation (Epic 6).** Recurring Entrate/Versamenti PAC are
  materialized on access up-to-today only, idempotent via a `regole_occorrenze`
  ledger, with skip = soft-delete + ledger mark (no scheduler, no Drift).
- **Honesty principle upheld in UI + engine.** Negative Saldo/Liquidità/Drift
  surfaced with sign (warm amber, never alarm-red, never clamped).

## 3. What was hard

- **Environment/toolchain saga (biggest cost).** WSL restarts repeatedly wiped
  `uv`/Node/bun. The `bun` on PATH was the **Windows** bun via WSL interop →
  corrupted workspace `node_modules` (`.exe` shims). Native bun could not be
  installed because the corporate **Cisco Umbrella** proxy does TLS-inspection on
  the GitHub asset CDN (`release-assets.githubusercontent.com`, root CA absent
  from the WSL trust store), failing the download. Resolution: **Node 22 + npm**
  locally (registry.npmjs.org + nodejs.org are not proxied), diverging from the
  bun-canonical repo. Significant time before any feature code.
- **Playwright e2e can't run locally** (same blocked CDN) → that gate relied on
  CI only.
- **biome version drift.** npm resolved `^2.3.14` to 2.5.0, whose stricter
  `suppressions/unused` flagged a clean file; fixed by pinning 2.3.14 (the
  bun.lock version) via `--no-save`.
- **Demo backend needed a manual restart per epic** (no `--reload`) to surface
  new endpoints; the running process kept old code while the DB had new schema.
- **One feature needed a product decision, not code:** the Secchiello
  cycle-advance-on-payment (Story 3.3) — "which Spesa is the payment?". Deferred,
  then implemented as an explicit **"Registra pagamento"** action rather than
  guessing auto-detection.

## 4. Lessons learned

1. **Verify the toolchain binary is native, not interop, before building** — and
   pin exact versions to the lockfile. Saved repeatedly once captured in memory.
2. **Pure calc + worked-example + purity-guard tests are worth it** — money math
   stayed correct and deterministic across the whole program with zero money bugs.
3. **Regenerate the typed client every backend change** and let the type-checker
   enforce the FE/BE contract; cheaper than runtime debugging.
4. **Defer ambiguous behaviour until the rule is explicit**, then implement it
   deliberately (the "Registra pagamento" action) — avoids encoding a wrong guess.
5. **CI is the source of truth for gates you can't run locally** (Playwright e2e);
   keep pushes small so a red gate points at a small diff.
